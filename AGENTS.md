## Build

Only CMake + Python `build.py`. No Makefile, no Cargo, no JS toolchain.

```bash
python build.py                        # CPU only (default)
python build.py --cuda --test -j$(nproc)   # CPU + CUDA
python build.py --vulkan --test -j$(nproc) # CPU + Vulkan (auto-compiles shaders)
python build.py --shader               # GLSLâ†’SPIR-Vâ†’C++ headers only (Vulkan prerequisite)
python build.py --rebuild              # clean + full rebuild
python build.py --clean                # rm -rf build/
```

Only one GPU backend at a time. C++23 required (GCC 14+ / Clang 18+ with libc++).

**If system GCC is too old** (e.g. Ubuntu 20.04 has GCC 9): install GCC 14+ from `ubuntu-toolchain-r/test` PPA, then build normally:
```bash
add-apt-repository -y ppa:ubuntu-toolchain-r/test
apt-get install -y g++-14
CC=gcc-14 CXX=g++-14 cmake -B build -DBACKEND_CPU=ON -DBUILD_TEST=ON
cmake --build build -j$(nproc)
```

**CUDA build** (requires CUDA Toolkit 12.8+; GCC as host compiler):
```bash
CC=gcc-14 CXX=g++-14 cmake -B build \
  -DCMAKE_CUDA_COMPILER=/usr/local/cuda-12.8/bin/nvcc \
  -DCMAKE_CUDA_HOST_COMPILER=/usr/bin/g++-14 \
  -DBACKEND_CUDA=ON -DBUILD_TEST=ON
cmake --build build -j$(nproc)
```

## Test

No test runner. Each `tests/*.cpp` compiles to its own executable:

```bash
./build/bin/test_tensor          # 9 tensor tests
./build/bin/test_dtype           # 6 dtype/F16/BF16 tests
./build/bin/test_ops_cpu         # 5 CPU op tests (add, mul, div, rms_norm, layer_norm)
./build/bin/test_page_attention  # 5 paged attention tests
```

Custom header-only test framework in `tests/test_utils.h` (ASSERT, ASSERT_NEAR, run_tests).
Integration executables: `build/bin/main`, `build/bin/genllm-cli`, `build/bin/benchmark`.

No lint, format, or typecheck tools exist. No CI.

## Architecture

- **Output**: `libgnellm.so` (shared library named `gnellm`). Test/link against it.
- **Backend ops**: CRTP-like with `device::dispatchOp` in `core/kernels.cpp`. Ops declared in `include/backend/{cpu,cuda,vulkan}/`, implemented in `src/backend/{cpu,cuda,vulkan}/`.
- **CUDA ops** are `.cu` files, compiled by NVCC with `cuBLAS` linkage.
- **Memory**: Three pools per device â€” weight (persistent), activation (reused per step), kv_cache (paged, per sequence).
- **Paged attention**: `PAGE_BLOCK_SIZE=16`, managed via `PagedAttentionManager` per-layer page tables.
- **Graph**: DAG via `OpFactory` DSL in `include/model/op_factory.hpp`. Tensor `src[]` edges, no memory at build time.
- **Tokenizer**: BPE loaded from GGUF metadata. Supports per-model pre-tokenization types.
- **Tests**: hardcoded model path `weights/Qwen3.5-0.8B-BF16.gguf` in `tests/main.cpp` â€” edit before run.

## Gotchas

- **CUDA backend** requires `CUDA_LAUNCH_BLOCKING=1` env var (or per-tensor sync) for activation pool correctness.
- **Only batch=1** (single-sequence autoregressive generation).
- **Only Qwen3/Qwen3.5** architectures implemented. Model factory (`include/model/model.h`) is extensible.
- **Vulkan shader rebuild**: after editing `shader/*/*.comp`, run `python build.py --shader` then rebuild.
- **No lockfiles or package manager**. Python scripts in `shell/` need `torch`, `transformers`, `numpy`, `pyyaml` installed externally.
- **Generated code**: SPIR-V C++ headers in `include/backend/vulkan/spv/*.h` â€” auto-generated by `python build.py --shader`.
- **Build type**: default `Release` (set in CMakeLists.txt). No debug/profile config in build.py.
- **No CTest integration** â€” tests are not wired to `ctest` despite `add_test()` calls.
- **`<span>` not transitively included**: `include/utils/tools.hpp` uses `std::span` without `#include <span>`. Works with libstdc++ (transitive), fails with libc++ â€” add the include if building with libc++.
- **`project()` CUDA language**: `CMakeLists.txt` project declaration must not unconditionally list `LANGUAGES C CXX CUDA`. CUDA language is now conditionally enabled via `enable_language(CUDA)` inside the `if(BACKEND_CUDA)` block â€” do not revert this.
- **CUDA + libc++ incompatibility**: CUDA 12.8 headers reject libc++ on x86 (`crt/host_defines.h`). Use GCC 14+ as host compiler with libstdc++ instead.
- **`std::format` with `std::vector`**: GCC 14 libstdc++ enforces `consteval` format-string checking. `std::format("{}", some_vector)` does not compile â€” vectors must be formatted manually (e.g. loop + `std::to_string`). Several source files needed this fix (`gguf_parser.cpp`, `graph.cpp`, `Qwen35.cpp`, `main.cpp`).
- **CUDA sigmoid missing**: `SigmoidImpl<Device::CUDA>` was not implemented in `src/backend/cuda/activation.cu` despite being dispatched in `kernels.cpp`. Added the kernel and explicit instantiation.
